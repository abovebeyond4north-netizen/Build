# Zero-Cost Revenue Learning Playbook

This playbook turns the dashboard into a safe learning system for ethical revenue experiments.

## Hard rules

- No paid tools.
- No spam.
- No deceptive claims.
- No private-data scraping.
- No external outreach without human approval.
- No automatic payments or money movement.
- No auto-merge.

## What the loop can do

Every scheduled run can:

1. Read the current revenue learning state.
2. Score possible no-cost experiments.
3. Pick the best next experiment.
4. Write a proposal report.
5. Update local learning state.
6. Let GitHub Actions test and build the project.
7. Open a pull request for review.

## What counts as a revenue experiment

A safe revenue experiment is a proposal that may help create value without spending money.

Examples:

- Improve this project as a portfolio proof asset.
- Prepare a GitHub Sponsors readiness checklist.
- Draft a service offer for human-reviewed outreach.
- Create a manual checklist for finding bounties or grants.
- Turn the dashboard into a reusable template roadmap.

## What does not count

The loop must not perform risky or external actions by itself.

It must not:

- send messages,
- create accounts,
- submit forms,
- make purchases,
- handle payments,
- promise guaranteed income,
- impersonate anyone,
- hide automation.

## Human review gate

The system can propose. A human decides.

That keeps the project useful, lawful, and trustworthy.
